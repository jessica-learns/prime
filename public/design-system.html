Poppins does the structural work in three additional weights: 800 for H1 headings, 500 for H3/H4 and labels, and 300 italic for subheads and body lightness. Poppins 900 and Instrument Serif never appear at the same size.

Key Rule

Poppins Black Commands, Instrument Serif Narrates

Poppins Black — When to Use
Hero headlines, cover titles, section titles, stat card numbers, and any text at 36px or above. Poppins Black's extreme weight and geometric letterforms create a bold, trustworthy presence. Always use at weight 900. Never below 28px — display weights need room to breathe.
Instrument Serif — When to Use
H2 subtitles, pull-quotes, card descriptions, and any editorial text between 18px and 32px. Always in italic. Its refined, classical letterforms contrast beautifully against Poppins Black's geometric weight and Poppins' geometric simplicity. Use at 400 weight only.

Three Voices Maximum
No text block should use more than three typographic voices — one display, one supporting, one body. In the example above: Poppins Black + coral Instrument Serif (display), Poppins 300 italic (supporting), and Poppins 400 (body). Instrument Serif italic should appear in only one color per text block — never stack a coral serif headline directly above a teal serif subtitle. If the headline uses Instrument Serif in coral, the subtitle drops to Poppins 300 italic in a neutral.

09
Rules

Design Principles

01
75/20/5 — Always
75% neutrals, 20% teal, 5% coral. If coral is appearing on more than labels, buttons, and accent words, something is wrong. Teal handles a lean middle layer — icon circles, secondary buttons, checkmarks. It should feel present but never dominant.
02
Poppins Black Commands, Instrument Narrates
Poppins at 900 weight, 36px and above. Instrument Serif italic between 18px and 32px. They never appear at the same size. Poppins 800 for H1, 500 for labels, 300 for body.
03
Never Two Alike in a Row
Alternate section backgrounds: White, Navy, Frost, Ink, Steel, Cloud. Every scroll should land on a surface change.
04
Contrast Is Law
All informational text requires a minimum color contrast of 4.5:1 for small text (<24px) and 3:1 for large text (24px+ or 19px+ bold). Silver and Ash never carry text on light surfaces. On dark surfaces, Ash is the body default — not Silver, not Pewter. Decorative elements like ghosted section numbers are exempt.
05
Clickable Means Contained
Every interactive element uses at least two affordance signals: container shape, distinct color, and a hover state. The only exception is underlined text links.
06
Italics Mean Something
In headlines, italic Instrument Serif connector words are prepositions — "for," "the," "and." In body text, Poppins 300 italic signals the one phrase worth remembering.
07
Asymmetry Over Symmetry
Default to 60/40 grids. Use tall-span cards to break monotony. Perfectly centered layouts feel generic; weighted layouts feel designed.
08
Three Voices Maximum Per Block
No text block should use more than three typographic treatments — one display, one supporting, one body. Instrument Serif italic may only appear in one color per block. If the headline uses coral Instrument Serif ("Datacenter Dominance"), the subtitle must drop to Poppins 300 italic in a neutral — never a second Instrument Serif in teal. Stacking two colored serifs makes the eye bounce instead of flow.
09
The Number Is the Ornament
Leading-zero Poppins Black numbers in display weight replace decorative icons. On light surfaces, numbers use Pewter or darker. On dark surfaces, Navy Light.
10
One Coral Per Section
Never mix Coral (#ED5E11), Coral Deep (#C4341A), Coral Mid (#FE6B18), and Coral Light (#FF8F50) in the same section or slide. Pick one and commit. Mixing coral values creates visual noise that reads as indecision, not variety. On dark surfaces, use Coral Light only. On light surfaces, use Coral or Coral Deep only. The accent is the accent — it works because there is exactly one.
10
Motion

Movement with Intention

Motion is never decorative — it communicates hierarchy, confirms actions, and guides attention. Every animation answers one question: "what just happened?" If it doesn't clarify something, remove it. The system uses four easing curves and five duration tiers, and every element follows predictable stagger rules so the interface feels composed rather than chaotic.

Four Easing Curves

Each curve has a role. Mixing them — using a spring where an expo belongs — feels as wrong as putting Coral on a secondary button. Hover over each track to preview the curve.

Out Expo
DEFAULT
The workhorse. Fast start, gentle landing. Use for entrances, reveals, page transitions, and any element moving into its resting position. Feels swift without being jarring.
cubic-bezier(0.16, 1, 0.3, 1)
Out Quart
SUBTLE
Softer deceleration. Use for hover states, color changes, opacity fades, and small positional shifts where the motion should feel effortless rather than decisive.
cubic-bezier(0.25, 1, 0.5, 1)
In-Out Quart
SYMMETRIC
Slow start, fast middle, slow finish. Use only for looping animations, loading states, and elements that move continuously. Never for user-triggered transitions.
cubic-bezier(0.76, 0, 0.24, 1)
Spring
ACCENT ONLY
Overshoots then settles. Use sparingly — toggle switches, success confirmations, and playful micro-interactions. More than two springs visible simultaneously feels amateurish.
cubic-bezier(0.34, 1.56, 0.64, 1)

Stagger Pattern — 60ms Steps

When multiple elements enter together — list items, cards, stat numbers — each one delays by 60ms after the previous. This creates a cascade that reads as composed rather than simultaneous. Cap at 6 items (360ms total delay); beyond that, batch into groups. Hover to preview.

Seven Motion Rules
01
Enter with Out-Expo, Exit with Out-Quart
Elements arriving should feel decisive (expo). Elements leaving should feel effortless (quart). This asymmetry mirrors how we pay more attention to what appears than what disappears.
02
Duration Maps to Distance
A button changing color (0px travel) gets 100ms. A card sliding 200px gets 350ms. A full-page transition gets 500ms. The eye needs proportionally more time to track longer journeys.
03
Stagger at 60ms, Cap at 6
Sequential element delays use 60ms steps. Beyond 6 items (360ms total), batch the remainder as a group — users stop perceiving individual entrances past that point.
04
Fade-Up Is the Default Entrance
Elements enter from 16–24px below their final position while fading from 0 to 1 opacity. This upward movement implies "rising to attention." Lateral entrances are reserved for navigation or slide-like transitions only.
05
Spring Is Coral-Level Rare
Like the coral accent at 5%, spring easing is the rarest motion — used only for toggle switches, success states, and playful micro-interactions. Two visible springs at once is the hard ceiling.
06
Respect prefers-reduced-motion
When the user's OS requests reduced motion, collapse all transitions to instant (0ms duration) and remove all transforms. Opacity changes may remain. Accessibility is not optional.
07
The Serif Word Arrives Last
In the mixed headline pattern, the italic Instrument Serif connector word fades in 120ms after the Poppins text. This delay reinforces the typographic hierarchy — the structural word establishes, the editorial word narrates.
CSS Custom Property Reference

All motion values are defined as CSS custom properties in :root. Use these tokens — never raw values — so the entire system can be tuned from one place.

/* Easing */
--ease-out-expo: cubic-bezier(0.16, 1, 0.3, 1);
--ease-out-quart: cubic-bezier(0.25, 1, 0.5, 1);
--ease-in-out-quart: cubic-bezier(0.76, 0, 0.24, 1);
